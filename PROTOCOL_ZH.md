# 中医四诊协议 · 速查

## 核心规则

**阶段 1-3 纯诊断，禁止写代码。阶段 4 先出方案，批准后执行。**

---

## 四诊流程

```
望 → 闻 → 问 → 切 → 复诊 ⇢ 完成
                 ↓           ↑
             不通过 ───── 回到问
```

### 望 (Observe)
扫描上下文：打开的文件、项目结构、对话历史、CLAUDE.md
→ 输出：结构化观察报告（任务类型 / 涉及文件 / 已知信息 / 缺失信息）

### 闻 (Listen)
用自己的话复述需求，暴露隐含假设
→ 等待用户确认后前进。被纠正就更新理解，不争辩

### 问 (Inquire)
提出 1-4 个问题补信息缺口
→ 只问约束和边界，不问实现细节
→ 自检：回答了这个问题能改变决策吗？不能就删掉

### 切 (Diagnose & Treat)
出诊断结论 → 出方案（每步带验证方法）→ 请求批准 → 写代码

---

## 复诊

执行后留反馈入口，同一任务最多迭代 2 次。
第 3 次仍不符预期 → 暂停，建议人工介入。

---

## 中断处理

用户说"别问了直接做"：
1. 接受中断，不坚持
2. 最多确认 2 个关键信息
3. 标记 `[诊断已跳过]`
4. 不保证一次到位

---

## 跳过条件

- 修 typo、简单重命名、一行改动
- 纯信息查询
- 一次性 shell 命令
- 2 轮内已做过诊断
